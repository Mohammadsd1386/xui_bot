# 🤖 ربات مدیریت پنل 3x-ui

ربات تلگرامی برای مدیریت کامل پنل 3x-ui از طریق تلگرام، مخصوص سرورهایی که به تلگرام دسترسی مستقیم ندارند.

---

## ✨ امکانات

| ویژگی | توضیح |
|-------|-------|
| ➕ افزودن کلاینت | ساخت کلاینت جدید با نام، حجم و مدت اعتبار |
| 📋 لیست کلاینت‌ها | مشاهده تمام کلاینت‌ها با وضعیت و حجم |
| ✏️ ویرایش کلاینت | تغییر حجم، مدت، فعال/غیرفعال کردن |
| 🗑 حذف کلاینت | حذف با تأییدیه |
| 📊 آمار مصرف | آمار کلی پنل |
| 🔄 ریستارت پنل | ریستارت سرویس Xray |
| ℹ️ وضعیت پنل | بررسی آنلاین بودن پنل |
| 🔒 فقط ادمین | فقط آیدی‌های تعریف‌شده دسترسی دارند |

---

## 📋 پیش‌نیازها

- Python 3.8+
- دسترسی به اینترنت از سرور (برای تلگرام)
- توکن ربات تلگرام از [@BotFather](https://t.me/BotFather)
- آیدی عددی تلگرام از [@userinfobot](https://t.me/userinfobot)

---

## 🚀 نصب و راه‌اندازی

### ۱. آپلود فایل‌ها روی سرور

```bash
# اگر از git استفاده می‌کنید
git clone <repo-url> xui-bot
cd xui-bot

# یا مستقیماً فایل‌ها را آپلود کنید
```

### ۲. تنظیم فایل .env

```bash
nano .env
```

مقادیر زیر را پر کنید:

```env
BOT_TOKEN=توکن_ربات_شما
PANEL_URL=http://آیپی-سرور:پورت
PANEL_PATH=/مسیر-پنل-شما
PANEL_USER=نام_کاربری_پنل
PANEL_PASS=رمز_پنل
INBOUND_ID=1
ADMIN_IDS=آیدی_عددی_تلگرام_شما
```

### ۳. اجرای اسکریپت نصب

```bash
chmod +x install.sh
bash install.sh
```

### ۴. اجرا

**اجرای مستقیم:**
```bash
source venv/bin/activate
python bot.py
```

**اجرا به عنوان سرویس سیستم (توصیه‌شده):**
```bash
chmod +x setup_service.sh
sudo bash setup_service.sh
```

## 🔧 دستورات مدیریت سرویس

```bash
systemctl status xui-bot     # وضعیت ربات
systemctl stop xui-bot       # توقف ربات
systemctl restart xui-bot    # ریستارت ربات
journalctl -u xui-bot -f     # مشاهده لاگ‌ها
```

## 📁 ساختار فایل‌ها

```
xui-bot/
├── bot.py           # منطق اصلی ربات تلگرام
├── xui_api.py       # ارتباط با API پنل 3x-ui
├── config.py        # خواندن تنظیمات
├── .env             # تنظیمات محرمانه (ویرایش کنید)
├── requirements.txt # کتابخانه‌های Python
├── install.sh       # اسکریپت نصب
└── setup_service.sh # اسکریپت سرویس systemd
```

---

## ⚠️ نکات مهم

- فایل `.env` را هرگز به اشتراک نگذارید
- `ADMIN_IDS` را حتماً پر کنید وگرنه همه می‌توانند از ربات استفاده کنند
- برای چند ادمین: `ADMIN_IDS=111111,222222,333333`
- `INBOUND_ID` معمولاً `1` است مگر اینکه چند inbound داشته باشید
